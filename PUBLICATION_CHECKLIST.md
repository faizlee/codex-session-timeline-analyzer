# Publication Checklist

Use this checklist before submitting the repository to the OpenAI Codex for Open Source form.

- [ ] Create a public GitHub repository.
- [ ] Push the files in this directory.
- [ ] Add 2-3 screenshots from `reports/sample.html` to the README or repository assets.
- [ ] Enable GitHub Actions by copying `ci/github-actions.yml` to `.github/workflows/ci.yml`, then confirm CI is green.
- [ ] Review `examples/sample-rollout.jsonl` for anonymization.
- [ ] Review generated reports before publishing screenshots.
- [ ] Add repository topics: `codex`, `openai`, `maintainer-tools`, `jsonl`, `developer-productivity`.
- [ ] Fill `OPENAI_OSS_APPLICATION_DRAFT.md` with the final public repository URL and current usage signals.

Recommended first public release tag: `v0.1.0`.
